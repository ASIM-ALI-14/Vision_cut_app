VisionCut

AI-powered video background remover for Android. Built with Jetpack Compose, ML Kit, RevenueCat, and AdMob — everything you need to ship a polished freemium app.

📱 Android 8.0+ 🎯 Target SDK 34 🔤 Kotlin 🎨 Jetpack Compose 🌙 Dark Mode Ready

Introduction

VisionCut is a production-ready Android application that uses on-device AI to remove the background from any video in real time. Users select a video, choose a background mode (green screen or transparent), optionally trim the clip, then export or share the result — all without sending data to a server.

The app is built on a fully documented, modular codebase using the latest Android best practices: Jetpack Compose UI, Coroutines + StateFlow for reactive state, RevenueCat for subscription management, and AdMob for monetization.

Installation

Step 1 — Open in Android Studio

  1. Extract the downloaded ZIP file to a folder on your machine.
  2. Open Android Studio → File → Open → select the extracted folder.
  3. Wait for Gradle to sync. This downloads all dependencies (may take a few minutes on first run).
  4. Once synced, connect an Android device or start an emulator (API 26+).
  5. Press the ▶ Run button. The app will build and install.
⚠️ Before running: The app requires AdMob and RevenueCat to be configured (see sections below). Running without them will cause a crash at launch.

Step 2 — Change Package Name

The default package name is com.devx.VisionCut. You must rename it to your own before publishing.

  1. In Android Studio, open app/build.gradle and update applicationId.
  2. Right-click the com.devx.VisionCut package in the project tree → Refactor → Rename → rename each segment.
  3. Update android:authorities in AndroidManifest.xml to match your new package name (used by FileProvider).
  4. Sync Gradle again after making changes.
app/build.gradle
android {
    defaultConfig {
        applicationId "com.yourcompany.yourappname"  // ← change this
        minSdk 26
        targetSdk 34
        versionCode 1
        versionName "1.0"
    }
}

AdMob Configuration

The app uses two ad formats: a Rewarded Interstitial ad on the "Remove Background" screen (free users must watch to process), and a Rewarded ad on the Export screen (to share or save).

1 — Create Your AdMob App

  1. Go to admob.google.com and sign in.
  2. Click Apps → Add App. Select Android, enter your app name.
  3. Copy the App ID (format: ca-app-pub-XXXXXXXX~YYYYYY).

2 — Add App ID to Manifest

AndroidManifest.xml
<application>
    <meta-data
        android:name="com.google.android.gms.ads.APPLICATION_ID"
        android:value="ca-app-pub-XXXXXXXX~YYYYYY" />  ← your App ID
</application>

3 — Create Ad Units

In AdMob, create two ad units for your app:

Ad UnitFormatFile to Update
Remove BG AdRewarded InterstitialInterstitialAdManager.kt
Export/Share AdRewardedRewardedAdManager.kt

4 — Replace Test IDs

Open each ad manager and replace the AD_UNIT_ID constant:

ads/InterstitialAdManager.kt
companion object {
    private const val AD_UNIT_ID = "ca-app-pub-XXXXXXXX/YYYYYY"  ← your real ID
}
ads/RewardedAdManager.kt
companion object {
    private const val AD_UNIT_ID = "ca-app-pub-XXXXXXXX/ZZZZZZ"  ← your real ID
}
✅ Test IDs are safe during development The current code uses Google's official test ad unit IDs. They show real ads from the test network and won't generate revenue or policy violations. Replace them only when you are ready to publish.

RevenueCat (Premium) Configuration

VisionCut uses RevenueCat to manage the premium subscription. The paywall UI is rendered by RevenueCat's official Compose SDK — you design it entirely in the RevenueCat dashboard, no code changes needed.

1 — Create a RevenueCat Project

  1. Sign up at app.revenuecat.com.
  2. Create a new project and add an Android app.
  3. Copy your Public SDK Key (starts with goog_ for Google Play).

2 — Set Your API Key

MyApp.kt
Purchases.configure(
    PurchasesConfiguration.Builder(
        context = this,
        apiKey  = "goog_your_real_key_here"  ← replace this
    ).build()
)
⚠️ Security: Do not commit your real API key to a public git repository. Store it in local.properties and read it via BuildConfig. See the API Key Security section for instructions.

3 — Create an Entitlement

  1. In the RevenueCat dashboard, go to Entitlements → New Entitlement.
  2. Set the Identifier to exactly: premium
    This must match the key used in PremiumViewModel.kt.
  3. Attach your Google Play subscription product to this entitlement.

4 — Design Your Paywall

The paywall UI is fully managed by RevenueCat's dashboard — no Kotlin code changes needed. Go to Paywalls in the RevenueCat dashboard and use the visual editor to set your pricing, images, and copy.

ℹ️ Entitlement key: If you want to rename "premium" to something else, update it in PremiumViewModel.kt in the handleCustomerInfo() function: customerInfo.entitlements["your_key"]

FileProvider Setup

FileProvider is used to safely share exported video files with other apps (WhatsApp, Gmail, etc.) on Android 7.0+. It is already pre-configured in the source — just make sure the authority matches your package name.

AndroidManifest.xml
<provider
    android:name="androidx.core.content.FileProvider"
    android:authorities="com.yourpackage.fileprovider"  ← must match package
    android:exported="false"
    android:grantUriPermissions="true">
    <meta-data
        android:name="android.support.FILE_PROVIDER_PATHS"
        android:resource="@xml/file_paths" />
</provider>
res/xml/file_paths.xml
<paths>
    <files-path name="internal" path="." />
</paths>

API Key Security

Keeping your RevenueCat API key out of version control is important. Here is the recommended approach:

  1. Add your key to local.properties (this file is git-ignored by default): 
    REVENUECAT_API_KEY=goog_your_real_key_here
  2. Read it in app/build.gradle: 
    def localProps = new Properties()
localProps.load(new FileInputStream(rootProject.file("local.properties")))

android {
    defaultConfig {
        buildConfigField "String", "RC_API_KEY",
            "\"${localProps['REVENUECAT_API_KEY']}\""
    }
}
  3. Use it in MyApp.kt: 
    apiKey = BuildConfig.RC_API_KEY

Colors & Theme

All app colors are defined in a single file: ui/theme/Color.kt. Change any value here and it updates everywhere automatically. Dark mode is fully supported — define a light and dark value for each color.

ui/theme/Color.kt — change your brand color
// ── Brand primary — change this one line to rebrand the app
val BrandPurple      = Color(0xFF6901FC)  // light mode primary
val BrandPurpleLight = Color(0xFF9747FF)  // dark mode primary

The theme is applied in MainActivity.kt via BGRemoverTheme. To disable dynamic color (Android 12+ wallpaper tint) and always use your brand color:

MainActivity.kt
BGRemoverTheme(dynamicColor = false) {   // false = always use BrandPurple
    AppNavigation(viewModel = viewModel)
}

Video Pipeline

VisionCut processes video entirely on-device. Here is how each frame flows through the pipeline:

Processing steps per frame
1. VideoFrameDecoder   — MediaCodec decodes MP4 → YUV frames → ARGB Bitmaps
2. SelfieSegmenter     — ML Kit produces a float confidence mask per pixel
                       Erode → Dilate → Gaussian blur (edge feathering)
3. FrameMaskApplier    — composites foreground bitmap onto chosen background
                       (solid color / transparent / blur / custom image)
4. VideoFrameEncoder   — uploads Bitmap → OpenGL texture → encodes via EGL surface
                       MediaMuxer writes H.264 to temp .mp4
5. AudioMuxer          — copies trimmed audio from original into final output

Background Modes

ModeOutput FormatUse Case
Green (SolidColor #00C853)MP4Chroma-key in video editors
TransparentMP4 with black fallback*Direct compositing

* MP4 cannot store alpha — transparent mode is converted to a black background in the output. To get true transparency, encode to MOV/WebM.

Debug Build

To run a debug build from Android Studio:

  1. Select the debug build variant in the Build Variants panel.
  2. Connect a device or start an emulator (API 26+, ideally a real device for ML Kit accuracy).
  3. Press ▶ Run.
ℹ️ In debug builds, Purchases.logLevel = LogLevel.DEBUG is enabled automatically. RevenueCat logs will appear in Logcat under the PremiumViewModel tag. This is disabled automatically in release builds.

Release Build

1 — Generate a Keystore

In Android Studio: Build → Generate Signed Bundle / APK → Android App Bundle → Create new keystore. Fill in the alias, passwords, and validity details. Store this file securely — losing it means you cannot update your app on the Play Store.

2 — Configure Signing in Gradle

app/build.gradle
android {
    signingConfigs {
        release {
            storeFile     file("path/to/your.keystore")
            storePassword "your_store_password"
            keyAlias      "your_alias"
            keyPassword   "your_key_password"
        }
    }
    buildTypes {
        release {
            signingConfig       signingConfigs.release
            minifyEnabled       true
            shrinkResources     true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
                          'proguard-rules.pro'
        }
    }
}

3 — Build the Release AAB

Build → Generate Signed Bundle / APK → Android App Bundle → release. Upload the resulting .aab file to the Google Play Console.

⚠️ Before publishing: Replace all test ad unit IDs with your real AdMob IDs. Replace the RevenueCat test key with your production key. Remove or change the debug API key comment in MyApp.kt.

ProGuard Rules

Add these rules to your proguard-rules.pro to prevent release build crashes:

proguard-rules.pro
# RevenueCat
-keep class com.revenuecat.purchases.** { *; }

# ML Kit
-keep class com.google.mlkit.** { *; }
-keep class com.google.android.gms.** { *; }

# AdMob
-keep class com.google.android.gms.ads.** { *; }

# MediaCodec / EGL (video pipeline)
-keep class android.media.** { *; }
-keep class android.opengl.** { *; }

# Kotlin Coroutines
-keepnames class kotlinx.coroutines.internal.MainDispatcherFactory {}
-keepnames class kotlinx.coroutines.CoroutineExceptionHandler {}

Frequently Asked Questions

The app crashes at launch after I open it in Android Studio

This almost always means AdMob or RevenueCat is not configured. Make sure you have added the AdMob App ID to AndroidManifest.xml and your RevenueCat API key to MyApp.kt, then clean and rebuild.

ML Kit segmentation is slow on my test device

ML Kit Selfie Segmentation in STREAM_MODE performs best on mid-range devices from 2020 onward. Processing speed is roughly 3–5x slower than real time on low-end devices. For testing, use a physical device rather than an emulator.

The rewarded ad never shows — just spins

During development, AdMob sometimes takes 30–60 seconds to serve the first test ad. Make sure you are using the official test ad unit IDs (already set in the source). If you switched to real IDs before getting AdMob approval, test ads won't serve.

RevenueCat always shows isPremium = false even after purchasing

Make sure the entitlement identifier in your RevenueCat dashboard is exactly premium (all lowercase, no spaces). This must match the key in PremiumViewModel.kt.

The exported video has no audio

Make sure READ_MEDIA_VIDEO (Android 13+) or READ_EXTERNAL_STORAGE (Android 12 and below) permissions are granted. The audio muxer reads audio directly from the original video file using MediaExtractor.

How do I change the app language / add translations?

All user-visible strings are hardcoded in the composables. To add localisation, move them to res/values/strings.xml and reference via stringResource(R.string.key). Create res/values-XX/strings.xml for each locale.

Can I add a custom background image instead of green screen?

Yes — FrameMaskApplier.BackgroundMode.Image(bitmap) is already implemented. Add a UI control in RemoveBGScreen.kt to let the user pick a photo, then pass it to viewModel.setBackgroundMode(Image(bitmap)).
